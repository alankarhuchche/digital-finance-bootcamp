# Premium Visual Review

## Purpose
Prevent generic diagrams and force premium visual quality. This gate ensures every visual on the site earns its place by genuinely teaching a concept, not merely decorating a page.

## When to use
- At the end of any phase that adds or modifies a custom visual component.
- Before committing a new visual to the main branch.
- When reviewing a visual after a significant design change.
- When a visual is added to an important topic page that a senior practitioner is likely to see.

---

## Pass criteria

A visual passes this gate when it:

- Has one clear job: it answers a specific question or teaches a specific distinction.
- Has a clear dominant teaching point that a reader sees within five seconds.
- Uses a distinct visual grammar: labels are short and specific, hierarchy is clear, active and inactive states are obvious.
- Active layers or elements are clearly differentiated from inactive or muted ones.
- Mode or scenario switching is obvious: the reader immediately understands what changed.
- Does not feel like a consulting slide, a generic boxes-and-arrows diagram, or a crypto gimmick.
- Matches or exceeds the Bank Route Map quality bar (the reference standard for this site).
- Is authored — it reflects a deliberate design decision, not a default template.
- Labels use banking-practitioner language: no generic tech labels, no hype terms.
- Does not rely only on colour to distinguish states (accessibility requirement).
- Works as a teaching asset at first glance: a reader could describe what it shows without reading the surrounding text.

---

## Fail criteria

A visual fails this gate when it:

- Uses equal-weight boxes connected by arrows with no hierarchy.
- Has more than eight visible labels at any one time without clear grouping.
- Has weak or absent hierarchy — all elements look the same weight.
- Has animation that runs constantly or loops without user interaction.
- Has animation that is decorative but does not help the reader understand a sequence or state change.
- Uses crypto gimmicks: spinning coins, chain icons, blockchain imagery for its own sake.
- Has poor contrast between active and inactive states.
- Shows active/inactive through colour alone without shape, weight or opacity difference.
- Uses labels that are too long to read quickly (more than four words per label).
- Does not change visibly enough when a mode or scenario is switched.
- Looks like it was auto-generated by a diagram tool.
- Would not improve understanding if removed and replaced with a good caption.

---

## Reference standard: Bank Route Map

The Bank Route Map is the current quality benchmark for this site. When evaluating a new visual, compare against it:

| Quality dimension | Bank Route Map standard |
|---|---|
| Layout | Asymmetric four-region grid, clear left-to-right flow |
| Active/muted state | Full opacity vs. ~0.25 muted; consistent across all elements |
| Mode switching | Scenario buttons with maturity pills; staged animation on switch |
| Labels | Short, banking-specific: "Faster Payments", "RTGS finality", "Nostro / vostro" |
| Hierarchy | Source ovals → control gates → route core → rail lanes → terminals → evidence badges |
| Animation | Sequential choreography; reduced-motion path; no looping |
| Mobile | Separate vertical strip with active nodes and arrows |
| Premium feel | Dark theme, amber/teal/purple maturity language, no generic icons |

A new visual should match or improve on this standard for its own concept.

## Review checklist

- [ ] State the visual's one job in one sentence. If you cannot, the visual has no clear teaching point.
- [ ] Describe the dominant element a reader sees first. Is that the right thing to see first?
- [ ] Count visible labels. More than eight active labels at once is a flag.
- [ ] Check active vs. inactive state contrast. Is it immediately obvious what is active?
- [ ] Check whether the visual uses only colour to show state, or also weight, opacity, shape or texture.
- [ ] Switch modes or scenarios rapidly. Does the visual reset cleanly? Is the new active state obvious?
- [ ] Check for looping animations. Anything that moves without user interaction is a flag.
- [ ] Read every label aloud. Are they all ≤4 words and banking-specific?
- [ ] Cover the surrounding text. Does the visual still teach something on its own?
- [ ] Compare to the Bank Route Map. Does this visual match or exceed its quality bar?
- [ ] Show the visual to someone unfamiliar with the page. Can they describe what it shows in 10 seconds?

## Example red flags

**Fail — boxes and arrows:**
> A horizontal flow diagram with seven equally-sized boxes labelled "Customer", "Bank A", "Scheme", "Network", "Bank B", "Ledger", "Confirmation", connected by arrows.

**Pass — layered state:**
> A vertical canvas with seven always-visible layers. Active layers illuminate in sequence; muted layers fade to low opacity but remain readable. Mode switching highlights different layer combinations.

---

**Fail — decorative animation:**
> A spinning token icon animates continuously alongside static text about DLT.

**Pass — instructive animation:**
> When Mode 2 (DLT same network) is selected, the token/shared-state layer activates with a brief shared-state glow, then the evidence badge fades in to show that custody and accounting still apply.

---

**Fail — poor label specificity:**
> Labels: "Input", "Process", "Network", "Output", "Record"

**Pass — specific labels:**
> Labels: "Customer ledger", "Payment message", "RTGS finality", "On-ledger token", "Correspondent settlement"

---

## Required report format

After completing this review, report:

1. Visual name and phase.
2. The visual's one job (one sentence).
3. Dominant teaching point assessment.
4. Active/muted state contrast assessment.
5. Label audit: count, length, specificity.
6. Animation audit: instructive or decorative, looping or not.
7. Mode/scenario switching assessment.
8. Comparison to Bank Route Map quality bar.
9. Any elements that fail and must be changed before passing.
10. Pass or fail verdict with reason.
